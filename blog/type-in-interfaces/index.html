Type in interfaces: a short field guide

Rules I follow, rules I break, and a cheat sheet for picking type that actually holds up across 100+ surfaces.

Typography in product interfaces is different from typography in print, different from typography on marketing sites, and different again from typography in editorial design. It has its own set of rules, its own failure modes, and its own aesthetic vocabulary.

Here's the condensed version of what I've learned after spending too many years thinking about it.

The rules I follow

One type scale, used everywhere. Pick your sizes once — say, 11, 12, 13, 14, 15, 16, 18, 22, 26 — and don't deviate. The discipline of a fixed scale makes inconsistency visible immediately. When something looks off, it's usually because someone reached for an off-scale size.

Weight over size for hierarchy. Before you make something bigger, try making it heavier. Going from 400 to 600 creates more legible hierarchy than going from 14px to 16px, and it takes up less space.

Line length is not optional. Every reading context has an optimal line length. For body copy, that's roughly 55–75 characters. For UI labels, it's whatever fits the container. But for anything someone is supposed to read, line length is a design decision, not a side effect.

Mono for numbers and data. Tabular numbers (monospaced figures) align vertically. In a list of timestamps or prices or counts, this matters. Use a monospaced font or tabular variant for anything that needs to scan vertically.

The rules I break

"Never use more than two typefaces." In a complex product with distinct domains — code editor, marketing page, data table, onboarding flow — two typefaces might not be enough. The rule exists to prevent chaos. If you understand why it exists, you know when to break it.

"Body copy should always be 16px." 16px is a reasonable default for longform reading. At 13px in a dense information interface, 16px reads as oversized and childlike. Match the size to the context, not to the rule.

The cheat sheet

Context Size Weight Line height
Display / hero 24–32px 600–700 1.2–1.3
Section header 13–15px 500–600 1.4
Body / label 13–15px 400 1.6–1.7
Caption / meta 11–12px 400–500 1.5
Code / mono 12–13px 400 1.6

These aren't universal laws. They're the ranges I've found work well in dense, data-heavy product interfaces. Marketing and editorial are different worlds.

The honest truth about typography in interfaces is that most decisions are made implicitly, through the defaults of whatever component library or design system you're working with. The designers who get good at it are the ones who make those decisions deliberately, even when they happen to match the defaults.
